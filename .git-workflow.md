# Git Workflow Guide

## Regular Commit Workflow

### Daily/Regular Commits

**When to commit:**
- After completing a task or feature
- At the end of a work session
- Before making major changes
- When you've made meaningful progress

**How to commit:**

1. **Check what changed:**
   ```powershell
   git status
   ```

2. **Add your changes:**
   ```powershell
   git add .
   ```

3. **Commit with descriptive message:**
   ```powershell
   git commit -m "Brief description of what you did"
   ```

4. **Push to GitHub:**
   ```powershell
   git push
   ```

**Or use the helper script:**
```powershell
.\commit-changes.ps1 "Description of your changes"
```

## Phase Milestones

### Creating a Phase Milestone

When you complete a phase, create a milestone tag:

```powershell
# After completing Phase 0
git tag -a phase-0 -m "Phase 0: Foundation Setup Complete"
git push origin phase-0

# After completing Phase 1
git tag -a phase-1 -m "Phase 1: Foundation Systems Complete"
git push origin phase-1

# And so on for each phase...
```

**Or use the helper script:**
```powershell
.\create-phase-milestone.ps1 0 "Foundation Setup Complete"
```

### Viewing Milestones

```powershell
# List all tags/milestones
git tag -l

# View details of a specific milestone
git show phase-0
```

### Available Phases

- **phase-0**: Foundation Setup
- **phase-1**: Foundation Systems
- **phase-2**: Core Gameplay Systems
- **phase-3**: World & Environment Systems
- **phase-4**: AI & Progression Systems
- **phase-5**: Content Systems
- **phase-6**: UI/UX Systems
- **phase-7**: Advanced Systems
- **phase-8**: Integration & Testing
- **phase-9**: Polish & Release

## Commit Message Guidelines

### Good Commit Messages:
- `"Added player movement controller"`
- `"Implemented inventory system UI"`
- `"Fixed bug in crafting recipe validation"`
- `"Updated technical spec for combat system"`
- `"Phase 1: Completed item database system"`

### Bad Commit Messages:
- `"changes"`
- `"fix"`
- `"update"`
- `"stuff"`

**Format:** Use present tense, be specific, keep it concise.

## Tips

1. **Commit often** - Small, frequent commits are better than large, infrequent ones
2. **Be descriptive** - Your future self will thank you
3. **Push regularly** - Don't let commits pile up locally
4. **Tag milestones** - Makes it easy to see project progress
5. **Review before committing** - Use `git status` and `git diff` to review changes
